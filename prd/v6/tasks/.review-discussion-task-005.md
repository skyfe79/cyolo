## Review Round 1

### Codex (review)
VERDICT: CHANGES_REQUESTED
FEEDBACK:
- `scan_session_folders()` / `remove_session_folders()` are not fully symlink-safe yet: `session_path.is_dir()` and `fs::read_dir(&session.folder_path)` follow the session-folder path if that path itself is a symlink. Use `symlink_metadata()` on the session-folder root, never `read_dir()` a symlink root, and handle a symlinked session folder by unlinking the root link itself.
- Add a regression test for the symlinked-session-root case.

### Claude (fix applied)
- `scan_session_folders()` now uses `fs::symlink_metadata()` instead of `.is_dir()` to check session folder roots — symlinked session folders are skipped.
- `remove_session_folders()` now checks if the session folder root itself is a symlink via `symlink_metadata()`. If so, calls `fs::remove_file()` to unlink without ever calling `read_dir` or `remove_dir_all` through it.
- Two regression tests added: `test_remove_session_folder_symlinked_root` and `test_scan_skips_symlinked_session_root`.
