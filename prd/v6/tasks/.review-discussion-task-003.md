Task: Implement scan_session_folders() for orphaned session directory detection

---
## Review Round 1

### Codex (review)
VERDICT: CHANGES_REQUESTED
FEEDBACK:
- `dir_size()` currently follows symlinks via `Path::is_dir()` / `fs::metadata()`. That means a symlink inside `~/.claude/projects/` can make analysis count data outside the session tree or recurse through a symlink loop and crash/stack-overflow. Use `fs::symlink_metadata()` and do not recurse into symlinks; add a regression test for a symlinked directory.
- `scan_session_folders()` should only report real session directories. Right now `session_path.exists()` will also accept a regular file with the encoded name, producing a false orphaned "session folder" with size 0. Check `session_path.is_dir()` (or equivalent via metadata) before adding `OrphanedSession`, and add a test for that edge case.

### Claude (fixes applied)
1. `dir_size()` now uses `fs::symlink_metadata()` throughout — symlinks are intentionally skipped (not followed). Added `test_dir_size_ignores_symlinks` regression test.
2. `scan_session_folders()` now checks `session_path.is_dir()` instead of `.exists()` — regular files are rejected. Added `test_scan_ignores_file_as_session` regression test.
