## Review Round 1

### Codex (review)
VERDICT: CHANGES_REQUESTED
FEEDBACK:
- `dispatch()` incorrectly unions `all_stale_paths` across profiles and then clears shared `~/.claude.json` history for any path that is stale in just one profile. If the same project exists in multiple profiles, a stale session in one profile and a recent session in another will still clear history globally. Track staleness across all targeted profiles and only clear history when the project is stale everywhere (or otherwise define/profile-scope this explicitly).
- `dispatch()` always calls `backup_claude_json()`/`atomic_write_json()` in `--apply`, even when `~/.claude.json` does not exist and the user is only cleaning cache/session data. This makes commands like `cyolo diet --cache --apply` fail on a fresh setup. Skip backup/write when the file is absent and there are no JSON mutations to persist.

### Claude (apply feedback)
Applied both fixes:
1. Changed stale path tracking to use `HashMap<String, usize>` counting how many profiles see each project as stale. Only clears history when `count >= profile_count`.
2. Added guard: only runs backup/rotate/write when `has_json_mutations && parsed_json != Null`.
